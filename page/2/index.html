DAY 1 FACESWAP TRY INSTALL

Today I have tried to install app to tried some function about faceswap

First：install faceswap app from https://github.com/deepfakes/faceswap/releases and install the Prerequisites Anaconda and git

As I have had git, so I just have to install the Anaconda:https://www.anaconda.com/download/.

Then you can let go, and follow the default options.

If you feel so slow, you can copy the http and go to xunlei to download. It will be faster.

PS: The faceswap app install is very slow and you shall wait for 10 minutes. If you are AMD like me, it may take you for 20 minutes at

least, and I also do not know if it can work. The word “desorted” was writen after it.

Shit, I am feeling freaking hard now. Writing the fucking note is really tired, but worthy.

先搞好虚拟环境，实际上换脸程序要在虚拟环境中进行，并且如果无法安装faceswap软件的话，也必须要在这个虚拟环境下才能安装进入对应的文件夹（自己创建的一个文件夹），用来存放仓库里面的文件，然后git clone对应仓库即可，然后就手动运行setup程序即python setup.py然后根据自己电脑的配置进行选择安装，这里不能无脑安装，比如说我是amd 用户，那么我就要在对amd时选择yes，如果我是NVIDIA gpu用户我就要选择NVIDIA gpu，选择完成之后就是漫长的等待，看看远方的视频吧家人们
对于pip –trusted-host 要用到正确的格式才行，在尝试到晚上四点仍然没有结果之后我直接睡觉，已经忘力昨天做了什么操作了，再次进行python faceswap.py gui这个命令的时候居然出现了不一样的错误，坠了，按照提示键入了两个命令
conda install tk
和
pip install -r ./requirements/requirements_amd.txt之后提示我的tensorflow版本不对
B:\faceswap\faceswap\ENV\python.exe -m pip install –upgrade pip -i http://mirrors.aliyun.com/pypi/simple/ –trusted-host mirrors.aliyun.com
What is tensorflow
奶奶的居然要重新setup一次·1
Couldn’t install tensorflow-cpu<2.11.0,>=2.10.0 with pip. Please install this package manually
Couldn’t install tensorflow-directml-plugin with pip. Please install this package manually
Faceswap必须使用tensorflow框架运行，如果提示No mudule named ‘tensorflow’
那么就是没有安装tensorflow，conda list 列出所有包看一下，果然没有tensorflow numpy等包
主要运用的命令就是
pip install 包名 -i http://pypi.douban.com/simple/ –trusted-host pypi.douban.com，而这个包名字跟-i中间必须要有一个空格否则会报错
ERROR: Cannot determine archive format of C:\Users\自己的用户名字\AppData\Local\Temp\pip-req-build-t35bzb_f
PS：cao，真的恶心
Pip install不起效果是网络延迟太大，
这时考虑换一个pip源，就是上面的做法
总的路程
在进行 python setup.py之后，如果仍然安装错误的话，下面进行手动安装，按照官方文档来
conda install tk
要注意看选项，我这边选择的是NVIDIA user 的选项，然后问system depen的话就是yes，不出所料这一步conda install tk也是有报错的的，接着输入命令
pip install -r ./requirements/requirements_nvidia.txt，再输入
python faceswap.py gui来尝试启动一下，如果提示缺少某个module的话就用上面的pip install 进行安装，如果提示没有tensorflow，那么用命令
conda install tensorflow==2.10.0，并且由于tensorflow要使用的是2.10版本的，所以必须要在后面加上==2.10.0，如果安装错误，上网找uninstall tensorflow的办法，然后重新安装即可。
PS：bug多多，记得检查自己每一步的步骤，语法有没有问题，最后实在不行了，再全部删掉重头再来，只需要把相关文件夹里面的资料删掉就好了，比如说我全部放在c盘的一个叫做faceswap的文件夹里面。
在提取图片的时候会出现错误，
OMP: Error #15: Initializing libiomp5md.dll, but found libiomp5md.dll already initialized

这个错误意思是有两个名字为libiomp5md.dll的图片被同时链接产生了错误，去ancada的安装文件夹里面搜索这个文件，把其中一个删掉即可
再次打开faceswap软件的方式：进入terminal之后，输入cd faceswap
然后输入python faceswap.py gui 即可
