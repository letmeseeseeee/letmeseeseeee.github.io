DAY 61

关于在navicat连接数据库时出现了1251 - Client does not support authentication protocol requested by server; consider upgrading MySQL client.
的解决办法

其实只是数据库的权限不对

找到数据库的安装位置

用命令进入（要打开cmd命令窗口输入）

cd C:\Program Files\MySQL\MySQL Server 8.0\bin

C:\Program Files\MySQL\MySQL Server 8.0\bin是数据库的默认安装路径

登录数据库

输入你的数据库的密码

接着输入以下两行代码
alter user ‘root‘@’localhost’ identified with mysql_native_password by ‘123456’;

这条SQL语句是用于在MySQL数据库中修改指定用户（在这里是‘root‘@’localhost’）的认证方式和密码

flush privileges;它的作用是将当前数据库的权限更改立即应用到MySQL服务器上

即可解决问题

dcl
即数据库的sql控制语法

感觉非常简单，只要不断重复就能很容易地记住

drop 是删除数据库/表/视图的权限

create是创建数据库/表

用户要读取数据库里面的数据必须有权限

mysql中有很多权限

all所有权限

select 查询数据的权限

insert 插入数据的权限

delete 删除数据的权限

alter 修改表修改字段的权限

drop 删除数据库/表/视图

视图在进阶篇中才有提及

create 创建数据库/表

通过dcl控制用户的权限的演示

查询权限

show grants for 用户名@主机名

授予权限

grant 权限列表 on 数据库名.表名 to 用户名@主机名

如果要给数据库所有的表赋予权限写数据库名.*即可

如果要指定数据库指定表名用指定数据库.指定表名即可

撤销权限即用revoke

revoke 权限列表 on 数据库名.表名 from 用户名@主机名

show grants for heima

接下来授予权限

授予有关键字grant all on 哪一个数据库哪一个表 to 哪一个用户？@哪一个主机？

注意revoke用的是from而grant用的是to

每次执行完成之后用show grants 查询一下权限

*代表的就是所有

ddl dml dql dcl 就是四种相关的sql语句
ddl
data definition language

dml
data manipulation language

dcl
data control language

dql
data query language
