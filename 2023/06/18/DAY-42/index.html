主函数和重载istream里面，输入int可能输入char

重载friend istream& operator>>(istream& cin, Card& input);这里输入的值可能是char而不是int

一个友元函数，重载了输入流操作中的输入流操作符’>>’，使其能够从输入流读取数据，并且将数据保存到card对象当中

该函数接受两个参数，一个是cin表示输入流对象，表示标准输入流

Card& input表示要读取数据的Card对象的引用

函数返回一个输入流对象(istream&)，允许在多个输入操作符连接在一起时进行链式输入

该函数的实现将根据Card对象的内部结构和输入格式，从输入流中提取必要的数据，并将其存储在Card对象的成员变量中

为什么要用友元函数？

为了更好地访问私有成员和保护成员 operator>>函数可以直接访问Card类的私有成员变量value，以读取和设置其值

具体的实现为：

friend istream& operator>>(istream& cin, Card& input);//重载从命令行输入

我的目的就是对以上重载的实现进行一个容错处理

ifstream与istream的区别

ifstream是istream的派生类，继承了istream的所有方法和操作符，并增加了一些用于从文件读取数据的特定方法

总结：istream是输入流的基类，提供了通用的输入功能，而ifstream是从文件中读取数据的特化输入流类。

这是一个重载输入流运算符，用于从输入流中读取数据并将其存储到Card对象的相应成员变量中

istream& operator>>(istream& cin, Card& input) {
try {
cout << “Please enter the name:\n”;
cin >> input.name_;

    cout << "Please choose the type (1. student, 2. teacher):\n";
    while (!(cin >> input.type_) || (input.type_ < 1 || input.type_ > 2)) {
        throw runtime_error("Invalid type! Please enter again: ");
    }
    
    cout << "Please enter the number (>0):\n";
    while (!(cin >> input.number_) || input.number_ < 0) {
        throw runtime_error("Invalid number (>0)! Please enter again: ");
    }
    
    cout << "Please choose the gender (1. male, 2. female):\n";
    while (!(cin >> input.gender_) || (input.gender_ < 1 || input.gender_ > 2)) {
        throw runtime_error("Invalid gender! Please enter again: ");
    }
    
    cout << "Please enter the place:\n";
    cin >> input.place_;
} catch (const exception& e) {
    // 捕获异常并处理错误消息
    cerr << "Exception caught: " << e.what() << endl;
    // 清除错误状态和输入流
    cin.clear();
    cin.ignore(numeric_limits<streamsize>::max(), '\n');
}

return cin;

}

以上代码使用try-catch块来捕获可能抛出的异常，当输入不满足要求的时候，通过throw语句抛出runtime_error类型的异常，异常消息作为参数传递

在catch块中，异常被捕获并处理，错误消息被输出到标准错误流(‘cerr’),并使用cin.clear()来清除输入流的错误状态标志

cin.ignore(numeric_limits::max(), ‘\n’)用于忽略剩余的输入内容。

这样，在发生异常时，会显示错误消息并清除错误状态，以便进行下一次输入。
